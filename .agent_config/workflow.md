# チームワークフロー (Team Workflow)

本プロジェクトは、PMBOKに準拠した以下の標準プロセスに従って進行する。
各フェーズの完了には、指定されたエージェントによる成果物のレビューと承認（Gate Check）が必須である。

## 0. Project Initiation (立ち上げ)
*   **主担当**: PM
*   **プロセス**:
    1.  PMがプロジェクトの目的・ゴールを定義。
    2.  `01_プロジェクトマネジメント計画書テンプレート.md` を作成し、QCDFの基準を確立。
*   **完了条件**: 計画書のユーザー承認。

## 1. Requirements Phase (要件定義)
*   **主担当**: Consultant
*   **プロセス**:
    1.  Consultantが`02_要件定義書テンプレート.md`を用いて要件定義書を作成。
    2.  PMが要件定義書をレビューし、スコープ記述書としての整合性を確認。
*   **完了条件**: ユーザー承認済みの要件定義書。

## 2. Basic Design Phase (基本設計)
*   **主担当**: Architect, UI/UX, Security
*   **プロセス**:
    1.  Architectが`04_基本設計書テンプレート.md`を用いてシステムアーキテクチャを決定。
    2.  UI/UXが`08_UIUXレビュー報告書テンプレート.md`に基づきデザイン方針を決定。
    3.  Securityが`07_セキュリティ監査報告書テンプレート.md`を用いて設計レビュー。
*   **完了条件**: 基本設計書の承認。

## 3. Detailed Design Phase (詳細設計)
*   **主担当**: Coder
*   **レビュー**: Architect
*   **プロセス**:
    1.  Coderが基本設計に基づき、`05_詳細設計書テンプレート.md`を作成。
    2.  クラス、メソッド、DB物理設計、API詳細仕様を記述。
    3.  Architectが整合性と実現可能性をレビュー。
*   **完了条件**: Architect承認済みの詳細設計書。

## 4. Environment Phase (環境構築)
*   **主担当**: DevOps
*   **プロセス**:
    1.  DevOpsが`03_インフラ構成書テンプレート.md`に基づき、開発・テスト環境を構築。
    2.  Docker/CI環境 (`run_command`使用) の動作確認。
*   **完了条件**: 再現可能な開発環境の確立。

## 5. Implementation Phase (実装)
*   **主担当**: Coder
*   **プロセス**:
    1.  Coderが詳細設計書に基づきコードを実装。
    2.  Coderが単体テストを実装・実行し、カバレッジ基準を満たすことを確認。
*   **完了条件**: ビルド可能で単体テストを通過するコード。

## 6. Verification Phase (検証)
*   **主担当**: QA, Security, UI/UX
*   **プロセス**:
    1.  QAが`06_テスト計画書テンプレート.md`に基づき、`browser_subagent`等を用いてテストを実行。
    2.  Securityが実装コードの脆弱性診断を実施。
    3.  UI/UXが実装画面のルック＆フィールを確認。
*   **完了条件**: テスト報告書での合格判定。

## 7. Deployment & Handover (納品・移行)
*   **主担当**: DevOps, PM
*   **プロセス**:
    1.  DevOpsが`09_運用マニュアルテンプレート.md`を作成・整備。
    2.  本番環境へのデプロイ（またはステージング確認）。
    3.  PMが全成果物を統合し、最終確認。
*   **完了条件**: ユーザー検収完了および運用マニュアルの引き渡し。

---

## 8. Kaizen Phase (自己進化)
**プロジェクト完了後、直ちにこのフェーズに移行する。**

*   **主担当**: CKO
*   **プロセス**:
    1.  **振り返り**: 全エージェントからのフィードバック収集。
    2.  **更新**: `10_KAIZENレポートテンプレート.md`を用いて改善点を分析し、テンプレート・ワークフロー・役割定義を更新。
    3.  **承認**: PM承認後、マスターリポジトリへ反映。
