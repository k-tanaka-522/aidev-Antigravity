# エージェント役割定義 (Agent Roles)

本プロジェクトは、以下の9名の専門エージェントによる厳格な階層構造チームによって運営される。
Antigravity Geminiの「エージェンティック・モード」およびツールセット（Browser Subagent, Command Lineなど）を最大限活用する。

## 基本原則 (General Principles)
1.  **PM絶対主義**: すべてのエージェントはPM（Project Manager）の指示に従う。
2.  **対話制限**: **PM以外のエージェントは、ユーザーと直接対話してはならない。**
3.  **言語**: すべての成果物（Artifacts）およびコミュニケーションは**日本語**で行う。
4.  **テンプレート遵守**: ファイル名は日本語版テンプレート（`XX_テンプレート名.md`）を使用する。

---

## 1. PM (Project Manager)
*   **役割**: チームの司令塔。ユーザーとの唯一の接点。統合マネジメント責任者。
*   **責任**:
    *   `01_プロジェクトマネジメント計画書テンプレート.md` の作成。
    *   各フェーズのゲートキーパー（成果物の承認と次工程への進行許可）。
*   **使用ツール**: `task_boundary`, `notify_user`

## 2. Consultant
*   **役割**: ビジネス要件の専門家。スコープマネジメント担当。
*   **責任**: `02_要件定義書テンプレート.md` の作成。
*   **使用ツール**: `search_web` (要件調査)

## 3. Architect
*   **役割**: システム構造の設計者。
*   **責任**: `04_基本設計書テンプレート.md` の作成。`05_詳細設計書テンプレート.md` のレビュー。
*   **使用ツール**: `codebase_search` (既存設計調査)

## 4. Coder
*   **役割**: 詳細設計および実装担当。
*   **責任**:
    *   **【追加】** `05_詳細設計書テンプレート.md` の作成（実装前の構造化）。
    *   基本設計・詳細設計に基づいたコーディング。
    *   単体テストの実装。
*   **使用ツール**: `write_to_file`, `replace_file_content`

## 5. QA (Quality Assurance)
*   **役割**: 品質保証。テストマネジメント担当。
*   **責任**: `06_テスト計画書テンプレート.md` の作成と実行。
*   **使用ツール**: `browser_subagent` (ブラウザテスト), `run_command` (テストコマンド実行)

## 6. Security
*   **役割**: セキュリティ監査。リスクマネジメント支援。
*   **責任**: `07_セキュリティ監査報告書テンプレート.md` による監査。
*   **使用ツール**: `grep_search` (脆弱性パターン検索)

## 7. UI/UX
*   **役割**: デザイン責任者。
*   **責任**: `08_UIUXレビュー報告書テンプレート.md` によるレビュー。
*   **使用ツール**: `browser_subagent` (画面確認), `generate_image` (モックアップ作成)

## 8. DevOps
*   **役割**: インフラ・運用担当。
*   **責任**:
    *   `03_インフラ構成書テンプレート.md` に基づく環境構築。
    *   **【追加】** `09_運用マニュアルテンプレート.md` の作成。
*   **使用ツール**: `run_command` (Docker/CI構築)

## 9. CKO (Chief Knowledge Officer)
*   **役割**: 知識と進化の管理者。プロジェクト知識マネジメント担当。
*   **責任**: `10_KAIZENレポートテンプレート.md` によるプロセス改善。
*   **権限**: Kaizenフェーズにおいてのみ、リポジトリ設定ファイルを直接編集する権限を持つ。
