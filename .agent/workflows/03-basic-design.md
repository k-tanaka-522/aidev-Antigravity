# 基本設計フェーズワークフロー

## 概要
IPA共通フレーム2013「2.3.1 システム方式設計」に準拠。

## 入力確認
`templates/01_要件定義/実成果物/` の成果物を読み込んでください。

---

## ⚠️ 重要：チーム分離の確認

基本設計に入る前に、以下を確認してください：

### アプリチーム/インフラチームの分離有無

**質問**: プロジェクトでアプリケーションチームとインフラチームは分かれていますか？

- **分かれている場合**:
  - BD009_IaC設計書を作成し、インフラ設計の責任分界を明確化
  - 成果物を `00_アプリアーキテクチャ/` と `01_インフラアーキテクチャ/` に分離
  - 詳細設計フェーズで並行作業が可能になります

- **分かれていない場合**:
  - 従来通り、すべての基本設計書を作成
  - 詳細設計でも単一チームで進行

参照: [team-separation-app-infra.md](../knowledge/documentation/team-separation-app-infra.md)

## 生成する成果物

### 出力先
`templates/02_基本設計/実成果物/`

チーム分離がある場合:
```
実成果物/
├── 00_アプリアーキテクチャ/    # アプリチーム担当
│   ├── BD001_システム方式設計書.md
│   ├── BD002_アプリケーション方式設計書.md
│   └── BD003_データベース方式設計書.md
│
└── 01_インフラアーキテクチャ/  # インフラチーム担当
    ├── BD004_ネットワーク構成設計書.md
    ├── BD005_セキュリティ設計書.md
    └── BD009_IaC設計書.md
```

チーム分離がない場合:
```
実成果物/
├── BD001_システム方式設計書.md
├── BD002_アプリケーション方式設計書.md
├── BD003_データベース方式設計書.md
├── BD004_ネットワーク構成設計書.md
├── BD005_セキュリティ設計書.md
├── BD006_運用設計書.md
├── BD007_移行設計書.md
├── BD008_テスト方針書.md
└── BD009_IaC設計書.md
```

## 必須成果物（IPA準拠）

テンプレートを使用して以下を作成してください：

### BD001_システム方式設計書_template.md
- システム全体の構成、技術スタック選定
- 非機能要件の実現方式

### BD002_アプリケーション方式設計書_template.md
- アプリケーションアーキテクチャ
- レイヤー構成、デザインパターン

### BD003_データベース方式設計書_template.md
- DB論理設計（ER図、正規化方針）
- データモデル

### BD004_ネットワーク構成設計書_template.md
- ネットワーク構成図
- VPC、サブネット設計

### BD005_セキュリティ設計書_template.md
- 認証・認可方式
- セキュリティ対策

### BD006_運用設計書_template.md
- 監視、バックアップ方針

### BD007_移行設計書_template.md
- データ移行計画（該当する場合）

### BD008_テスト方針書_template.md
- テスト戦略、品質基準

### BD009_IaC設計書_template.md **（重要）**
- Terraform/CloudFormation選定理由
- スタック分割方針
- State管理方針
- **チーム分離がある場合は必須**
