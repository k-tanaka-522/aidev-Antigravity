## 🚀 使用方法

### 1. サブシステムディレクトリの作成

プロジェクトのサブシステムに応じてディレクトリを作成します：

```bash
mkdir -p "実成果物/01_ユーザー管理"
mkdir -p "実成果物/02_注文管理"
mkdir -p "実成果物/03_決済管理"
mkdir -p "実成果物/04_共通基盤"
```

### 2. テンプレートのコピー

親ディレクトリのテンプレートをコピーして使用します：

```bash
# クラス設計書の作成例
cp ../DD001-01_クラス設計書_template.md \
   "実成果物/01_ユーザー管理/DD001-01_クラス設計書_認証モジュール.md"

# 単体テスト仕様書の作成例
cp ../DD004_単体テスト仕様書_template.md \
   "実成果物/01_ユーザー管理/DD004_単体テスト仕様書_UserService.md"
```

### 3. プレースホルダの置換

コピーしたファイル内の以下のプレースホルダを実際の値に置換します：

- `{プロジェクト名}`
- `{サブシステム名}`
- `{モジュール名}`
- `{YYYY/MM/DD}`
- `{作成者名}`

## ✅ 文書作成のチェックリスト

各文書を作成する際は、以下を確認してください：

### DD001-01: クラス設計書
- [ ] 1ファイルのクラス数は10以下
- [ ] クラス図は5-10クラス程度
- [ ] ページ数は50ページ以下
- [ ] 単一のモジュール/パッケージに絞られている

### DD004: 単体テスト仕様書
- [ ] テスト対象クラスは3つ以下
- [ ] テストケース数は50以下
- [ ] ページ数は30ページ以下
- [ ] 正常系・異常系・境界値テストを含む
