# 02. ディレクトリ構成定義

## 1. 構成方針

本プロジェクトのディレクトリは、**「エージェント制御情報（.agent）」**と**「開発成果物（docs, src）」**に明確に分離される。

## 2. 各ディレクトリの役割

### `.agent/`（エージェント制御情報）

エージェントシステムの動作を規定する「知識」と「ルール」を格納するディレクトリである。利用者はこの内容を編集することで、エージェントの振る舞いを調整できる。

| ディレクトリ | 役割 | 格納内容 |
|:---|:---|:---|
| **`workflows/`** | 作業手順定義 | 開発プロセス全体の手順書。エージェントはこれに従って作業を進める。（例：`full_dev_process_ja.md`） |
| **`personas/`** | 役割定義 | 各エージェントの専門性・責務を定義したファイル。（例：`consultant.md`, `coder.md`） |
| **`standards/`** | 品質基準・規約 | 成果物が遵守すべき技術基準。`tech/app`（アプリ）、`tech/infra`（インフラ）等に分類される。 |
| **`templates/`** | 成果物テンプレート | 各種ドキュメントの雛形。エージェントはこれを埋める形で成果物を作成する。 |
| **`guides/`** | 補足ガイドライン | 各フェーズの詳細な作業ガイド。ワークフローから参照される補足資料。 |

### `docs/`（ドキュメント成果物）

エージェントが生成するドキュメント成果物を格納するディレクトリである。日本のSI開発で一般的な成果物体系に準拠する。

| ディレクトリ | 格納内容 |
|:---|:---|
| **`01_要件定義/`** | 要件定義書、機能一覧、非機能要件定義書 等 |
| **`02_基本設計/`** | システムアーキテクチャ設計書、API定義書、ER図、インフラ構成図 等 |
| **`03_詳細設計/`** | クラス図、シーケンス図 等（必要に応じて作成） |
| **`00_System_Design/`** | 本エージェントシステム自体の設計書（本ドキュメント群） |

### `src/`（ソースコード成果物）

エージェントが生成するソースコードを格納するディレクトリである。

| ディレクトリ | 格納内容 |
|:---|:---|
| **`backend/`** | サーバーサイドコード（Python, Go, Node.js 等） |
| **`frontend/`** | クライアントサイドコード（React, Next.js 等） |
| **`infra/`** | IaCコード（Terraform, CloudFormation） |

### `test_reports/`（検証エビデンス）

品質検証の証跡を格納するディレクトリである。

| ディレクトリ | 格納内容 |
|:---|:---|
| **`browser/`** | ブラウザ自動操作による検証ログ、スクリーンショット、録画 |
| **`unit/`** | 単体テスト実行結果レポート |
