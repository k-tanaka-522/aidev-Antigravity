# 02. ディレクトリ・スキーマ定義 (Directory Schema)

## 1. 構造の意図
本プロジェクトのディレクトリは、**「AI脳 (.agent)」** と **「成果物 (docs, src)」** に明確に分離される。

## 2. ディレクトリ詳細

### `.agent/` (The Brain)
エージェントシステムを駆動するための「知識」と「ロジック」が格納される場所。
ユーザーはこの内容を編集することで、エージェントの振る舞いを調整（チューニング）できる。

| パス | 役割 | 格納されるもの |
| :--- | :--- | :--- |
| **`workflows/`** | **アルゴリズム** | 開発プロセスの手順書。エージェントにとっての「メインプログラム」。(例: `full_dev_process_ja.md`) |
| **`personas/`** | **人格定義** | 各エージェントのシステムプロンプト拡張定義。(例: `consultant.md`, `coder.md`) |
| **`standards/`** | **法律・規約** | 成果物が守るべき品質基準。`tech/app`, `tech/infra` 等に分類される。 |
| **`templates/`** | **金型** | 成果物の空フォーマット。エージェントはこれを埋める形で出力を行う。 |
| **`guides/`** | **参考書** | プロセスごとの詳細ガイドライン。ワークフローから参照される補足資料。 |

### `docs/` (The Documents)
エージェントが生成する「ドキュメント成果物」。
日本の一般的なSI開発（ウォーターフォール）の成果物体系に準拠する。

| パス | 内容 |
| :--- | :--- |
| **`01_要件定義/`** | 要件定義書、機能一覧、非機能要件など。 |
| **`02_基本設計/`** | アーキテクチャ設計、API定義、ER図、インフラ構成図など。 |
| **`03_詳細設計/`** | クラス図、シーケンス図など（必要に応じて作成）。 |
| **`00_System_Design/`** | **(本ドキュメント)** このエージェントシステム自体の設計図。 |

### `src/` (The Code)
エージェントが生成する「ソースコード成果物」。

| パス | 内容 |
| :--- | :--- |
| **`backend/`** | サーバーサイドコード (Python/Go/Node.js 等)。 |
| **`frontend/`** | クライアントサイドコード (React/Next.js 等)。 |
| **`infra/`** | IaCコード (Terraform/CloudFormation)。 |

### `test_reports/` (The Evidence)
品質保証の証拠。

| パス | 内容 |
| :--- | :--- |
| **`browser/`** | Browser Agentによる操作ログ、スクリーンショット、動画。 |
| **`unit/`** | 単体テスト結果レポート。 |
